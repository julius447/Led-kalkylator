# VOL.2 PLAN

## Summary
Vol.2 turns the LED-kalkylatorn from a polished-but-inert prototype into a working lead magnet by closing the three doctrine-breaking gaps in priority order: (1) a user-adjustable electricity price so the hero kr/år becomes THE user's defensible number — the dossier's named #1 uncertainty and the single largest honesty/trust win; (2) a real value-then-ask lead-capture flow behind the currently-dead primary CTA, with the computed scenario attached so sales gets warm context; and (3) a shareable result loop (scenario-in-URL + OG meta + visual delningskort) that makes the BRF "vår förening sparar X kr/år" moment reproduce itself in styrelsegrupper. Around those, vol.2 hardens correctness (calc error boundary, NaN/clamp fixes, a real structural data validator), fixes the WCAG blockers (aria-live scope, slider aria-valuetext, teal contrast, 44px targets, H1), and sharpens the squint-test (4xl hero, grouped trio, collapsed breakdown) — all without inflating a single number.

## CONFLICTS RESOLVED
- elpris override depth: Research/Component lenses want full component decomposition (spot/påslag/skatt/elnät) while UX/PM/Dev want a single kr/kWh total slider. RESOLVED: ship the single total-kr/kWh override now (must-ship #1, S, satisfies the honesty blocker immediately and reconciles with the existing breakdown row); push component decomposition + elnät-only override + reducerad energiskatt to nice-to-have. The total slider is the doctrine MUST; the decomposition is refinement.
- Zone dropdown — replace vs supplement: UX finding suggests replacing the zone SELECT with a slider; Copywriting/architecture need the zone for geo-CTA logic and for seeding the default price. RESOLVED: keep the zone SELECT (it seeds the slider default and drives geo/CTA), ADD the kr/kWh slider as a first-class draggable input on top. Zone selection resets the slider to that zone's default; once the user drags, their value wins.
- Lead form vs 'one primary CTA per section' doctrine: Lead-magnet wants a form; design doctrine forbids two primaries. RESOLVED: the form REPLACES/EXPANDS the CTA in place (the button reveals the form), not a second primary button. Share stays a ghost button.
- Breakdown open-by-default: it's the transparency centerpiece (honesty) but force-open buries the hero on mobile (UX/spacing). RESOLVED: collapse by default with a teasing summary — transparency is preserved one click away, satisfying both the squint-test and the honesty moat.
- CTA teal contrast vs brand color: a11y needs the teal darkened to pass AA, brand wants the bright Ampy teal. RESOLVED: darken teal only where it's TEXT or the CTA fill (rgb(0,124,107)/rgb(0,118,101)); keep the bright --action-primary for non-text accents (slider accent-color, 1px borders) which are WCAG-exempt — palette still reads as Ampy.
- Geo granularity: PM/architecture flag that bidding zones aren't service areas and want postnummer-based geo. RESOLVED: defer real geo to nice-to-have; the kr/kWh slider reduces the zone's load-bearing role, and default_servicezon:true is an acceptable documented placeholder for vol.2 so the full number always shows.
- Share card now vs real OG pipeline: Marketer wants dynamic per-result OG images (server-side, L); others want a screenshottable card now. RESOLVED: ship the in-DOM visual card + static OG fallback meta + scenario-in-URL hydration in must-ship; defer the dynamic per-result OG-image endpoint to nice-to-have (needs a backend).

## HONESTY GUARDRAILS
- No email/lead wall before the answer — the hero kr/år and full breakdown must be visible on load and stay visible; the lead form only appears AFTER the value, triggered by the CTA (value-then-ask).
- Conservative defaults stay conservative — zone seeds the LOW price default; the kr/kWh slider only lets users move toward their real (often higher) bill, never auto-inflates. The honesty copy must say real savings are likely higher, not lower.
- Fix the false statement, don't keep it: the 'Elpriser och antaganden kan justeras ovan' line must become true the moment the elpris control ships, or be reworded to exactly what the tool does.
- Transparent math preserved — the breakdown formula uses the user's own values and the live kr/kWh override; collapsing it by default is fine but it must remain one click away with no values hidden.
- CO2 only for Företag/BRF, never Privat, and only ever as marknadsbaserad nordisk residualmix (ESG/Scope 2) — never framed as the consumer's physical avoided emissions; flag the figure for re-verification (Ei 2025 due now).
